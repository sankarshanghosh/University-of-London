Accessibility:

When creating my website, I wanted to make sure that it was accessible to all users, regardless of their abilities or assistive technologies they may be using. Here are three ways in which my site is accessible:

  1. Semantic HTML: I made sure to use semantic HTML throughout my website. This means that I used specific HTML tags to indicate the structure and purpose of the content on each page. By doing this, I have made it easier for screen readers and other assistive technologies to understand the content of the page. For example, I used heading tags to indicate the hierarchy of the content on the page and list tags to structure lists.
  2. ARIA landmarks: Another way in which my site is accessible is through the use of ARIA landmarks. These HTML attributes define regions of a webpage, such as the header, navigation, main content, and footer. By using ARIA landmarks, I have helped users navigate and understand the structure of the webpage, especially when using a screen reader.
  3. Alt text for images: I made sure to add alt text to all images on my website. Alt text provides a textual description of images for people who are visually impaired or using a screen reader. I used descriptive and concise alt text to convey the purpose or content of each image.
  4. Skip navigation link: Lastly, I added a skip navigation link to my website. This link allows users to skip repetitive navigation links and jump straight to the main content of the page. This feature is especially useful for users who are navigating with a keyboard or screen reader.

By implementing these accessibility features, I hope to make my website as inclusive and accessible as possible to all users, regardless of their abilities or assistive technologies they may be using.

Usability:

Here are 3 ways in which I considered the usability of my site.

  1. Clear Navigation: The website has a clear and consistent navigation structure. The menu is easy to find and use and is present on every page. The navigation bar clearly highlights the active page the user is currently on which makes it very easy for a user to understand which section of the website they are currently accessing. The website is designed to ensure that visitors can easily find the content they are looking for.
  2. Responsive Design: The website is optimized for different screen sizes, ensuring that visitors can view and use the website on any device, including desktops, laptops, tablets, and mobile phones. This is achieved through the use of a responsive design, which adapts the layout of the website to fit the screen size of the device being used.
  3. Dark Mode/Light Mode Functionality: The website has a dark mode and light mode feature, which allows users to customize their viewing experience. Users can switch between dark and light modes using a button located in the header. This feature was implemented using JavaScript and CSS, and it uses local storage to save the user's preference. This feature enhances the usability of the website by making it easier for users to read content in their preferred mode.

Evaluation II:

Looking back on my work, I believe that there are two areas that could have been improved. The first area is the mobile optimization of the website. While the website looks great on desktop, it's not so great on mobile or tablet devices. In hindsight, I realize that I should have considered the mobile experience of the website from the start and used media queries and other CSS functionalities to optimize the website for all devices.

The second area that could have been improved is the articles section. Currently, when users click on an article link, they are redirected to an external website. This is not ideal as it takes users away from my website. In the future, I plan to have the articles show content directly within my website. This would allow users to stay on my website and provide a better user experience.

If I were to do things differently next time, I would prioritize mobile optimization and internal article content from the start of the project. By doing so, I would be able to deliver a more polished and user-friendly website to my audience.
